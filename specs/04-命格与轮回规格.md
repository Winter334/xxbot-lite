## 2. 当前已拍板前提

根据前序讨论，04 必须遵守以下前提：

- 命格是角色的长期差异来源之一。
- 命格需要有“开卡味”“牌面感”“晒面板价值”。
- 一期不做多系统叠加，命格必须保持轻量。
- 命格可以影响 **PVE 与 PVP**，不做模式区别对待。
- 境界仍然是主线成长轴，命格不能压过境界与本命法宝。
- 命格主要通过“数值偏移”制造差异，不通过复杂规则改写来制造差异。
- 命格名称本身需要体现品质感；高品质命格应当一眼看出更有牌面。
- 命格池允许“名字绑定品质”，不要求某个效果骨架必须铺满全部品质版本。
- 一期目标是尽快落地，不为了未来扩展而做额外元系统。

## 3. 命格系统定位

### 3.1 核心定位

命格在一期中的职责是：

- 提供角色开局差异
- 提供长期小幅成长偏向
- 提供面板展示价值与虚荣心反馈
- 为轮回重修提供核心驱动力

一句话：

> **命格是角色的“天生底子与牌面标签”，不是一整套 build 系统。**

### 3.2 设计目标

命格应当做到：

- 抽到高品质命格时，玩家会想截图、想晒
- 低品质命格也不至于完全没用
- 命格能影响实战与成长，但不应翻转境界压制关系
- 命格能驱动“我要不要重修赌新命”的选择

### 3.3 非目标

命格不是：

- 技能系统
- 规则特判系统
- 连锁构筑系统
- 多阶段养成系统
- 收藏图鉴系统

## 4. 一期边界

### 4.1 一期只做 3 类命格

一期命格只允许落在以下 3 类：

1. **修行型**
   - 作用：提升挂机修为收益

2. **战斗型**
   - 作用：提升角色基础三维中的一项或多项
   - 允许影响：
     - 通天塔战斗
     - 论道榜 / PVP 战斗

3. **机缘型**
   - 作用：提升通天塔首通时“额外掉落判定”的成功率

### 4.2 命格效果的实现原则

一期命格只允许做：

- 百分比数值加成
- 固定字段驱动
- 统一结算逻辑内的轻量乘算 / 加算

不允许做：

- 复杂条件触发
- 回合触发技
- 行为规则改写
- 独立命格脚本
- 命格专属战斗机制

### 4.3 命格如何影响 PVP

一期命格明确影响 PVP。

原因：

- 命格本来就是角色长期差异的一部分
- 前期主要仍由境界与法宝决定强弱
- 到同境界争锋阶段，命格应当自然参与胜负差异
- 不应做“PVE 生效、PVP 失效”的割裂规则

## 5. 命格类型与效果骨架

### 5.1 修行型

修行型命格只允许提供：

> **挂机修为 +X%**

实现建议：

- 在挂机收益最终结果上做一次小倍率修正
- 不改变挂机周期
- 不改变挂机时长上限
- 不改变补算时机

### 5.2 机缘型

机缘型命格只允许提供：

> **通天塔首通额外掉落率 +X%**

实现建议：

- 先按通天塔正常首通规则发放固定奖励
- 再执行一次“额外掉落判定”
- 机缘型命格提高该判定的触发概率

说明：

- 一期额外奖励池可以非常轻，初期只放额外器魂包也可以
- 后续若扩展洗炼材料、法宝材料，也应优先接入同一额外奖励池
- 因此机缘型命格保留“额外掉落率”这一表述，不硬绑死在器魂单材料上

### 5.3 战斗型

战斗型命格不使用“登塔战力 +X%”或“总战力 +X%”这种黑盒写法。

战斗型命格只允许直接作用于：

- 杀伐
- 护体
- 身法

允许的组合形式：

1. **单项型**
   - 杀伐
   - 护体
   - 身法

2. **双项型**
   - 杀伐 + 护体
   - 杀伐 + 身法
   - 护体 + 身法

3. **全项型**
   - 杀伐 + 护体 + 身法

说明：

- 这属于“战斗型内部的三维组合”，不属于跨类别双效果命格
- 一期不允许出现“战斗 + 修行”“战斗 + 机缘”这类跨类别复合命格

## 6. 稀有度与抽取权重

### 6.1 稀有度档位

一期固定 4 档：

- **普通**
- **稀有**
- **史诗**
- **传说**

### 6.2 抽取权重

一期命格抽取权重正式采用：

- **普通：60%**
- **稀有：25%**
- **史诗：10%**
- **传说：5%**

### 6.3 品质感原则

命格名称应体现品质感：

- 普通：名字更朴素、直白、平实
- 稀有：开始带锋芒、异象、神兽或气势词
- 史诗：明显更玄、更狠、更有宿命感
- 传说：一眼顶级，名字应明显更有牌面

说明：

- 不强求字数一致
- 不强求都为 4 字名
- 名称本身就是品质感的一部分

## 7. 数值档位与分配规则

### 7.1 通用档位

一期命格数值档位正式采用：

- **普通：3%**
- **稀有：6%**
- **史诗：9%**
- **传说：12%**

### 7.2 修行型与机缘型

修行型与机缘型直接使用完整档位：

- 普通：`+3%`
- 稀有：`+6%`
- 史诗：`+9%`
- 传说：`+12%`

### 7.3 战斗型的分配规则

战斗型命格中的 `3 / 6 / 9 / 12` 应理解为：

> **该命格的总强度档位**，而不是每个属性都吃满档位。

为保证平衡，一期正式采用：

#### 单项型

单项属性吃满档位：

- 普通：`+3%`
- 稀有：`+6%`
- 史诗：`+9%`
- 传说：`+12%`

#### 双项型

双项平分总档位：

- 普通：每项 `+1.5%`
- 稀有：每项 `+3%`
- 史诗：每项 `+4.5%`
- 传说：每项 `+6%`

#### 全项型

三项平分总档位：

- 普通：每项 `+1%`
- 稀有：每项 `+2%`
- 史诗：每项 `+3%`
- 传说：每项 `+4%`

### 7.4 平衡意图

该设计应形成：

- **单项型**：最专精
- **双项型**：更均衡
- **全项型**：最泛用

其中：

- 全项型不应成为无脑最优解
- 单项型应在对应方向上最有压迫感
- 双项型应承担“稳定、好用”的中间定位

## 8. 获取流程

### 8.1 创建角色

玩家创建角色时：

- 系统随机获得 **1 条命格**
- 不做新手手动挑选
- 不做免费无限刷初始

### 8.2 命格抽取建议实现

由于命格池条目数与各品质内条目数可能不同，建议采用两段式随机：

1. 先按稀有度权重抽取品质
2. 再从该品质对应的命格条目池中随机 1 条

这样做的原因：

- 不会因为某品质条目多，就歪掉整体爆率
- 允许后续继续往某品质补命格条目，而不影响大盘权重

### 8.3 获得播报

一期仅在以下场景全服播报：

- **玩家获得传说命格**

说明：

- 普通、稀有、史诗命格不做全服播报
- 传说命格需要明显提供“抽到了就想晒”的爽感

## 9. 轮回重修

### 9.1 系统定位

轮回重修是一期命格系统的唯一改命出口。

作用：

- 让玩家有机会赌更好的命格
- 为高品质命格提供稀缺感
- 为命格系统提供长期博弈动力

### 9.2 轮回频率

一期正式采用：

> **每天可进行 1 次轮回重修**

建议实现为：

- 按 `Asia/Shanghai` 自然日刷新
- 每日 00:00 重置轮回次数

不建议使用 24 小时滚动 CD。

### 9.3 轮回结果

玩家轮回后：

- 重新随机获得 **1 条新命格**
- 命格不保证与上次不同
- 一期不做保底
- 一期不做“避免重复命格”保护

### 9.4 轮回代价

一期轮回代价正式采用：

> **丢失所有当前主线进度**

应当重置：

- 当前境界
- 当前修为
- 当前塔层进度
- 当前挂机进度相关状态
- 当前本命法宝成长
- 当前论道即时排名状态

### 9.5 轮回后保留内容

轮回后保留：

- Discord 账号身份
- **轮回次数**
- **历史最高塔层**
- **历史最高论道名次**

说明：

- 一期不做命格历史
- 一期不做命格图鉴
- 一期不做收藏记录

### 9.6 轮回与播报

若轮回后获得传说命格，仍按正常规则进行全服播报。

## 10. 初始命格池设计原则

### 10.1 池子结构原则

一期命格池采用：

> **名字绑定品质，但效果骨架不要求全档覆盖。**

也就是说：

- 某种效果可以只有普通与史诗版本
- 某种效果也可以拥有完整 4 档
- 一期重点是“池子看起来丰富”，而不是机械地做全排列

### 10.2 数量建议

一期初始命格条目总量建议控制在：

- **12 ~ 18 条**

当前推荐的首版实现可先落在：

- **约 18 条**

并满足：

- **战斗型条目数多于修行型与机缘型**

### 10.3 命名分层建议

可参考以下命名方向：

- **普通**：偏炉火、江风、草木、山石、泉水、泽光等凡俗自然意象
- **稀有**：偏龙虎、震雷、玄武、青鸾、坎水等异象 / 灵兽 / 卦象意象
- **史诗**：偏七杀、扶摇、红尘、太虚、天禄等更强的命理 / 宿命 / 玄门意象
- **传说**：偏混元、乾元、紫微、大梦等宏大气数意象

命名语感应尽量接近：

- 命书批语
- 命数断词
- 星宿判词
- 卦象断语

而不是：

- 技能名
- 体质名
- 装备名

说明：

- 一期不强求所有命格都带 `命 / 格 / 局 / 天命` 后缀
- 若名字本身已足够像“批命断语”，可以直接使用四字命名
- 重点是让玩家一眼感觉这是一种“被判出来的命数”，而不是某种主动能力

## 12. 数据结构建议

### 12.1 FateDefinition（命格定义表）

建议字段：

```text
fate_key
fate_name
rarity                 // normal | rare | epic | legendary
category               // cultivation | combat | fortune
affected_stats         // 例如 atk / def / agi / idle_cultivation / first_clear_bonus_drop
value_basis_points     // 例如 300 = 3%
flavor_text
broadcast_on_obtain    // bool
is_easter_egg          // bool
enabled                // bool
sort_order
created_at
updated_at
```

说明：

- `value_basis_points` 建议使用整数，避免浮点误差
- 双项 / 全项型可使用多个受影响字段，或使用结构化 JSON 字段表达
- 后续只需继续加配置，不需要改底层规则

### 12.2 Character 侧相关字段

建议最小字段：

```text
fate_key
reincarnation_count
last_reincarnated_on   // YYYY-MM-DD（按 Asia/Shanghai）或等价自然日标记
```

如已存在角色主表，也可直接并入角色表，不强制单独拆表。

## 13. 展示与文案边界

### 13.1 面板展示

角色面板中，命格应处于较高展示优先级。

建议至少展示：

- 命格名称
- 命格品质
- 命格效果一句话摘要

### 13.2 品质感要求

高品质命格的名字、文案、播报都应更有牌面。

尤其传说命格：

- 名称要明显更顶
- 面板中应更醒目
- 获得时可配合结构化播报模板

### 13.3 彩蛋命格

一期允许存在：

- **1 条彩蛋型传说命格**

原则：

- 不需要独立规则
- 不需要独立掉落方式
- 正常进入传说池即可
- 重点在“另类但仍然值得晒”

---

## 14. 非目标 / 禁止实现

在 04 范围内，以下内容默认禁止：

- 命格图鉴
- 命格历史记录
- 命格收集册
- 命格升级
- 多命格槽
- 命格融合
- 羁绊系统
- 命格专属技能
- 回合触发技
- 气机上限 / 恢复改写
- 突破规则改写
- 为命格单独做脚本引擎或 DSL

一句话：

> **04 只做“有牌面、有差异、能驱动重修、实现很轻”的命格系统，不做第二套大养成。**
