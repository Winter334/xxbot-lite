## 2. 当前已拍板前提

根据当前讨论，12 必须遵守以下前提：

- 宗门系统做成 **轻量标签系统**，而不是重管理系统。
- 玩家加入宗门后，最重要的变化是：
  - 获得宗门标签
  - 参与宗门资源争夺
  - 参与宗门贡献结算
- 宗门职位只做 **标签**，不带管理权限。
- 宗主默认为创建者；副宗主 / 长老等按贡献自动排序。
- 世界资源点应当是 **跨境界都能参与** 的玩家互动玩法，而不是纯高战力互殴玩法。
- 宗门奖励可以同时发：
  - `灵石`
  - `器魂`
  - `修为`
- `灵石` 作为新的通用资源 / 货币，现在就可以引入。
- `灵石` 在当前阶段可以 **弱用途甚至暂无用途**，但字段与产出可以先落地。
- 现有悬赏系统在保留器魂奖励的前提下，额外增加灵石奖励。
- 资源点不再是固定常驻点，而是 **每日随机刷新、存在 3 天后枯竭消失** 的动态点位。
- 不额外设计全服资源点活跃上限，依靠 **每日刷新量 + 3 天寿命** 自然形成稳定循环。
- 每名玩家同一时间只能绑定 **1 个资源点岗位**，切换目标点时自动从旧点移除。
- 资源点争夺按 **宗门** 结算，而不是把多个进攻宗门揉成一个总攻方。
- `输运` 不再参与攻防胜负，只负责提高资源点产出。
- 每名输运成员为所在资源点提供 `+10%` 产出加成，**单点不设上限**。

## 3. 系统定位

### 3.1 核心定位

轻量宗门系统是一条建立在现有角色、阵营、公开面板之上的 **群体归属与公共争夺玩法**。

它的职责是：

- 给玩家提供比“单人挂机 / 单人登塔 / 单人论道”更明显的群体归属
- 提供一个不论高低境界都能参与的公共互动玩法
- 为灵石资源提供自然产出入口
- 让阵营之外再多一层“我属于谁”的身份感

一句话：

> **宗门系统负责把“个人修仙”扩展成“有标签、有归属、有共同目标”的轻量群体玩法。**

### 3.2 非目标

当前版本的宗门系统不是：

- 大型 MMO 公会系统
- 权限管理系统
- 复杂内政系统
- 多页面运营后台
- 多资源库存管理系统

## 4. 设计原则

### 4.1 轻量原则

一期正式采用：

- 宗门只承担：
  - 标签
  - 成员归属
  - 资源争夺
  - 结算奖励
- 不承担复杂管理

### 4.2 低门槛参与原则

资源争夺必须满足：

- 高境界玩家有优势
- 低境界玩家也能稳定贡献
- 不允许“只有前排大号能玩，其他人纯围观”

### 4.3 与现有系统共存原则

宗门系统应尽量复用：

- 现有角色面板
- 现有阵营标签
- 现有轻量交互模式
- 现有每日自然日重置口径

## 5. 宗门基础规则

### 5.1 宗门数量与归属

一期正式采用：

- 每个角色同一时间只能加入 **1 个宗门**
- 一个宗门可包含多个成员

### 5.2 创建宗门

一期建议：

- 玩家达到 `筑基` 后可创建宗门
- 创建宗门不消耗额外资源
- 创建者自动成为宗主

说明：

- 用境界门槛控制泛滥创建
- 当前阶段不引入建宗材料

### 5.3 宗门名称

一期建议：

- 长度限制：`2 ~ 12`
- 禁止换行
- 禁止 `@everyone` / `@here`
- 宗门名称全服唯一

### 5.4 成员上限

一期建议：

> **单个宗门成员上限为 30 人。**

原因：

- 防止玩家全部挤进一个宗门
- 让资源争夺更容易形成多方竞争
- 与当前轻量系统规模更匹配

### 5.5 加入与退出

一期正式采用：

- 加入宗门为 **自愿行为**
- 退出宗门也为 **自愿行为**
- 不做审批，不做踢人

### 5.6 冷却规则

一期建议：

- 退出宗门后，`24 小时` 内不可加入新宗门
- 加入宗门满 `24 小时` 后，才可领取宗门日结算奖励

原因：

- 防止玩家来回跳宗门蹭奖励
- 保持宗门归属感

### 5.7 宗门解散

一期建议：

- 当宗门最后一名成员退出时，宗门自动解散

## 6. 宗门职位标签

### 6.1 职位设置

一期建议保留以下标签：

- `宗主`
- `副宗主`
- `长老`
- `弟子`

### 6.2 标签规则

一期正式采用：

- `宗主`：创建者
- `副宗主`：除宗主外，近 7 天贡献第 1 名
- `长老`：除宗主 / 副宗主外，近 7 天贡献前 3 名
- 其他成员：`弟子`

### 6.3 权限规则

一期正式采用：

> **所有职位都只是标签，不携带额外权限。**

说明：

- 这样能保留牌面感
- 不会把系统拖进复杂管理逻辑

### 6.4 宗主继承

一期建议：

- 若宗主退出宗门：
  - 自动由当前贡献最高成员继承宗主标签
- 若无其他成员：
  - 宗门解散

## 7. 宗门与阵营联动

### 7.1 宗门阵营字段

一期正式采用：

> **每个宗门都带一个 `faction_key`。**

取值：

- `neutral`
- `righteous`
- `demonic`

### 7.2 宗门阵营来源

一期建议：

- 宗门的阵营属性继承创建者当前阵营

也就是说：

- 正道玩家创建的宗门是正道宗门
- 魔道玩家创建的宗门是魔道宗门
- 中立玩家创建的宗门是中立宗门

### 7.3 加入限制

根据当前共识，一期正式采用：

- 已有阵营的玩家，只能加入 **所属阵营** 的宗门
- 中立玩家可加入任意宗门

### 7.4 中立玩家规则

一期正式采用：

- 中立玩家加入非中立宗门时，**不自动改变自身阵营**
- 阵营仍由原有阵营系统单独决定

说明：

- 这是当前版本的轻量妥协
- 后续若发现中立过于占便宜，再补更严格联动

## 8. 灵石资源

### 8.1 资源定位

一期正式采用：

> **灵石作为新的通用资源 / 货币引入。**

### 8.2 当前阶段职责

当前阶段灵石主要承担：

- 宗门资源争夺奖励
- 悬赏额外奖励
- 后续系统的扩展接口

### 8.3 当前阶段不承担的职责

一期不做：

- 灵石购买属性
- 灵石直接购买器魂
- 灵石商城
- 灵石背包道具

### 8.4 存储建议

一期建议在 `Character` 上直接新增：

- `lingshi`

不挂在 `Artifact` 上。

## 9. 世界资源点

### 9.1 核心定位

世界资源点仍然是宗门系统的主要互动玩法，但不再是固定常驻点。

它的职责是：

- 提供每天都可能变化的公共争夺目标
- 给不同宗门制造明确的攻守关系
- 提供灵石、器魂、修为奖励
- 让高低境界玩家都能在同一套玩法里找到岗位

### 9.2 一期资源点种类

一期正式采用 3 种资源点：

#### 1. 灵矿

- 主奖励：`灵石`
- 副奖励：少量 `器魂`

#### 2. 魂脉

- 主奖励：`器魂`
- 副奖励：少量 `灵石`

#### 3. 灵泉

- 主奖励：`修为`
- 副奖励：少量 `灵石`

### 9.3 刷新规则

一期改为：

- 资源点不再是固定的 `5` 个常驻点
- 每个自然日随机刷新一批新资源点
- 一期建议每日刷新 `2` 个资源点
- 刷新类型按 `灵矿 : 灵泉 : 魂脉 = 2 : 2 : 1` 的权重抽取

说明：

- 每日刷新 `2` 个、每点存在 `3` 天时，全服同时活跃的资源点通常会稳定在 `6` 个左右
- 这样既比旧版更有变化，也不会把面板撑得过于零散

### 9.4 寿命与枯竭

一期正式采用：

- 每个资源点自刷新日起存在 `3` 天
- 到第 `4` 天时，资源点枯竭并从活跃列表移除
- 资源点移除时，不保留占领进度，也不保留驻守名单

### 9.5 是否需要全服活跃上限

一期正式采用：

> **不额外设计全服活跃上限。**

原因：

- 每日刷新量已经可控
- `3` 天寿命会自然形成“新点出现、旧点消失”的循环
- 少一套总量兜底规则，系统更直观

## 10. 资源争夺规则

### 10.1 结算周期

一期正式采用：

> **按自然日结算一次资源点收益。**

这里的“日结算”只处理奖励发放与刷新 / 枯竭，不重置仍在存续中的资源点攻防状态。

### 10.2 实现方式

一期继续优先采用：

> **懒结算**

也就是：

- 跨天后第一次发生宗门相关交互时
- 先结算上一日仍存续资源点的奖励
- 再补刷新当日新资源点
- 最后清理达到寿命上限的旧资源点

说明：

- 已存续资源点的占领进度、驻守名单、输运名单不因跨天自动重置
- 只有资源点易主、枯竭或玩家切换目标点时，相关状态才会变化

### 10.3 参与次数

一期正式采用：

- **不单独设计宗门行动次数**
- 每次宗门行动消耗 `1` 点 `气机`
- 气机不足时不可进行宗门行动

说明：

- 资源点玩法继续吃现有气机体系
- 不新增第二套体力或挑战券

### 10.4 行动类型

一期保留 3 类行动，但职责重做如下：

#### 1. 强争

- 定位：进攻向
- 只有非占有宗门成员可以使用
- 使用后，角色绑定到该资源点的本宗进攻名单
- 每次点击都会触发一次宗门级对抗结算

#### 2. 护持

- 定位：防守向
- 只有当前占有宗门成员可以使用
- 使用后，角色绑定到该资源点的防守名单
- 护持成员会持续计入之后的防守战力，直到主动切换到别处

#### 3. 输运

- 定位：后勤向
- 只有当前占有宗门成员可以使用
- 使用后，角色绑定到该资源点的输运名单
- 输运不参与攻防胜负，只提高该点最终产出

### 10.5 玩家绑定规则

一期正式采用：

- 每名玩家同一时间只能绑定 **1 个资源点岗位**
- 岗位取值为：`强争` / `护持` / `输运`
- 玩家对新资源点执行相关行动时，自动从旧资源点的名单中移除
- 这种切换不额外收取冷却时间，只正常消耗这次操作的 `1` 点气机

说明：

- 这样能避免“每个资源点点一下就都占了坑”
- 人力分配会真正成为宗门需要取舍的内容

### 10.6 宗门级对抗原则

一期正式采用：

> **资源点争夺按宗门结算，不按单个玩家结算。**

也就是说：

- 发起操作的是某个玩家
- 真正参与胜负判定的是该玩家所属宗门在该资源点上的当前总兵力
- 若当前资源点已有归属，则攻方宗门对守方宗门做一次宗门级对抗判定

### 10.7 宗门级对抗不等于多人战斗

一期明确不做：

- 多角色逐回合战斗
- 多宗门同时上场的复杂战报
- 器灵 / 神通 / 法宝词条 / 命格 / 标题 / 游历增益等额外战斗效果介入资源点争夺

一期正式采用：

> **宗门级对抗只读取双方参与成员的“基础战力”，然后走一套轻量判定函数。**

这里的“基础战力”建议定义为：

- 只取角色当前境界 / 小阶段对应的基础三维
- 不叠加器灵、法宝词条、命格、称号、游历增益等额外效果
- 可沿用现有战力折算权重：
  - `base_power = base_atk * 1.35 + base_def * 1.5 + base_agi * 1.15`

### 10.8 为什么不按“谁高一点谁必赢”

一期正式采用：

> **细微战力差距不应直接锁定结果。**

原因：

- 如果攻方只比守方多 `1` 点战力就能稳定必赢，体感会很差
- 资源点争夺需要有“相近时会拉扯”的味道

### 10.9 建议判定口径

一期建议采用“差距大直接判，差距小进摇摆区”的轻量函数：

1. 先计算：

```text
attack_power = 攻方宗门当前绑定在该点强争名单中的基础战力总和
defense_power = 守方宗门当前绑定在该点护持名单中的基础战力总和
ratio = attack_power / max(defense_power, 1)
```

2. 再判定：

- 当 `ratio >= 1.30` 时，攻方直接胜
- 当 `ratio <= 0.70` 时，攻方直接负
- 当 `0.70 < ratio < 1.30` 时，进入摇摆区

3. 摇摆区内攻方胜率建议：

```text
win_rate = clamp(0.5 + (ratio - 1.0) * 0.5, 0.35, 0.65)
```

说明：

- 这样战力接近时不会固定输赢
- 差距明显时仍然有明确强弱感
- 这比硬接一套完整战斗系统更轻，也更符合资源点玩法的节奏

### 10.10 占领进度

一期正式采用：

- 无主资源点：占领进度目标为 `2`
- 已有归属的资源点：占领进度目标为 `4`

说明：

- 无主点应更快形成首个归属
- 有主点应更强调“攻下来”和“守住”的过程感

### 10.11 无主资源点规则

一期正式采用：

- 无主资源点不进行攻防判定
- 非占有宗门成员执行 `强争` 时，直接为本宗增加 `1` 点占领进度
- 谁先达到无主点的占领目标，谁就成为新占有宗门

### 10.12 已占有资源点规则

一期正式采用：

- 对已占有资源点执行 `强争` 时，先触发一次宗门级对抗判定
- 若攻方胜利，则该攻方宗门的占领进度 `+1`
- 若攻方失败，则本次不增加占领进度
- 防守方不单独累加“防守进度”，其优势体现在更高的防守战力与持续驻守

### 10.13 多个进攻方的处理

一期正式采用：

> **多个进攻方必须按宗门分开结算。**

具体规则：

- 每个进攻宗门都有自己独立的占领进度
- 不能把多个宗门的进攻成员合并成一个总攻方
- 守方始终只有当前占有宗门
- 最先达到占领目标的进攻宗门获得该资源点

### 10.14 易主后的状态变化

一期建议：

- 某宗门占领进度达标后，资源点立即易主
- 旧占有宗门的护持名单与输运名单清空
- 其他进攻宗门的占领进度清空
- 新占有宗门当前仍绑定在该点 `强争` 名单中的成员，自动转为该点 `护持` 名单
- 新占有宗门的输运名单初始为空，需玩家后续自行投入

### 10.15 输运规则

一期正式采用：

- `输运` 不参与攻防胜负
- `输运` 只影响资源点最终结算产出
- 每名输运成员为该资源点提供 `+10%` 产出加成
- 单点不设产出加成上限

说明：

- 这样输运不再和强争 / 护持抢同一条数值逻辑
- 低境界玩家可以通过后勤位稳定产生价值

### 10.16 个人宗门贡献

一期建议保留个人宗门贡献系统，用于：

- 宗门职位标签排序
- 每周贡献排行
- 个人参与反馈

建议口径：

- `强争`：`60 + global_stage_index * 3 + 随机 0~15`
- `护持`：`50 + global_stage_index * 2 + 随机 0~10`
- `输运`：`40 + 随机 0~5`

额外建议：

- 若某次 `强争` 在已占有资源点上打赢宗门级对抗，可额外追加少量贡献奖励
- 这样既保留行动反馈，也让“打穿一次防线”更有手感

---

## 11. 资源点奖励

### 11.1 奖励发放原则

一期正式采用：

- 资源点奖励按 **资源种类** 决定
- 奖励按 **当前占有宗门** 发放
- 资源点最终奖励要乘上该点当前的输运产出加成

### 11.2 输运加成公式

一期正式采用：

```text
output_multiplier = 1 + 0.1 * transport_member_count
final_reward = floor(base_reward * output_multiplier)
```

说明：

- `transport_member_count` 为该点当前输运名单人数
- 单点不设上限

### 11.3 奖励领取资格

一期建议：

> **结算时，仍对宗门资源点有有效驻守 / 输运绑定，或当日做出过至少 1 次资源点行动的成员，视为可领取成员。**

这样能兼容两种情况：

- 玩家当天主动行动
- 玩家前一天已投入人手，今天仍持续驻守或输运

### 11.4 灵矿奖励建议

每占有 `1` 个灵矿，符合资格成员的基础奖励为：

- `灵石 +200`
- `器魂 +10`

最终发放时再乘输运加成。

### 11.5 魂脉奖励建议

每占有 `1` 个魂脉，符合资格成员的基础奖励为：

- `器魂 +18`
- `灵石 +120`

最终发放时再乘输运加成。

### 11.6 灵泉奖励建议

每占有 `1` 个灵泉，符合资格成员的基础奖励为：

- `修为 +当前阶段上限的 5% + 当前大境界固定修为`
- `灵石 +100`

当前大境界固定修为建议表：

| 大境界 | 固定修为 |
|---|---|
| 炼气 | `10` |
| 筑基 | `40` |
| 结丹 | `200` |
| 元婴 | `1000` |
| 化神 | `5000` |
| 炼虚 | `20000` |
| 合体 | `80000` |
| 大乘 | `200000` |
| 渡劫 | `400000` |

灵泉最终奖励建议流程：

1. 先算出基础修为奖励
2. 再乘输运加成
3. 最终仍受当前阶段剩余可得修为上限约束

### 11.7 为什么灵泉按固定值 + 百分比发修为

一期正式采用：

> **灵泉奖励按“固定修为 + 当前阶段上限百分比”共同发放。**

原因：

- 纯固定值会导致低境界爆炸、高境界没感觉
- 纯百分比又会让低境界体感过低
- 固定值 + 百分比叠加，可以同时覆盖低境界体感与高境界成长性

### 11.8 个人贡献额外奖励

一期建议：

- 每日宗门内贡献前 `3` 名，额外获得：
  - `灵石 +100`

这样可以强化参与感，但不做过重奖励差距。

## 12. 悬赏系统追加灵石奖励

### 12.1 当前问题

当前悬赏讨伐只发：

- 器魂
- 气运
- 善名

这会让悬赏奖励整体过于偏向法宝线。

### 12.2 新口径

一期正式采用：

> **悬赏讨伐在保留器魂奖励的前提下，额外发放灵石奖励。**

### 12.3 奖励公式建议

一期建议：

```text
reward_soul = target.bounty_soul
reward_lingshi = reward_soul * 10
```

说明：

- 不新增 `bounty_lingshi`
- 灵石奖励直接从现有悬赏值导出
- 这样实现最简单，调参也方便

### 12.4 为什么不新开第二套悬赏金额

原因：

- 一期没必要维护两套悬赏累积值
- 现有悬赏榜、悬赏成长、悬赏清零逻辑都已围绕 `bounty_soul`
- 在 payout 时派生灵石奖励即可

## 14. 数据结构建议

### 14.1 新增表建议

一期建议新增：

#### `sects`

- `id`
- `name`
- `faction_key`
- `founder_character_id`
- `created_at`
- `updated_at`

#### `world_resource_sites`

- `id`
- `site_key`
- `site_name`
- `site_type`
- `owner_sect_id`
- `spawned_on`
- `expires_on`
- `settlement_day`
- `state_json`

说明：

- `site_key` 不再代表固定常驻点编号，而是动态资源点实例标识
- `spawned_on` / `expires_on` 用于支持动态刷新与 `3` 天寿命

### 14.2 `state_json` 建议内容

一期建议至少承载：

```json
{
  "attack_progress": {
    "12": 2,
    "18": 1
  },
  "attack_members": {
    "12": [101, 102],
    "18": [205]
  },
  "guard_member_ids": [301, 302],
  "transport_member_ids": [401, 402, 403]
}
```

说明：

- `attack_progress` 以宗门为单位记录占领进度
- `attack_members` 以宗门为单位记录当前强争名单
- `guard_member_ids` 只记录当前占有宗门的护持名单
- `transport_member_ids` 只记录当前占有宗门的输运名单

### 14.3 Character 字段建议

当前 `Character` 已有：

- `sect_id`
- `sect_joined_at`
- `sect_last_left_at`
- `sect_contribution_total`
- `sect_contribution_weekly`
- `sect_contribution_daily`
- `sect_last_contribution_on`
- `lingshi`

为了支持跨点切换，建议额外补充：

- `sect_bound_site_id`
- `sect_bound_site_role`

原因：

- 玩家切换资源点时，需要快速知道自己原先绑定在哪个点
- 如果完全靠扫描所有资源点的 `state_json` 找人，后续会越来越重

### 14.4 为什么不单独做职位表

一期正式采用：

> **职位标签按贡献实时计算，不单独落库存储。**

原因：

- 职位无权限，只是展示标签
- 直接按贡献计算更省事

## 15. 实现顺序建议

一期建议按以下顺序落地：

1. 调整 `WorldResourceSite` 结构，支持动态刷新、寿命与新的 `state_json`
2. 在 `Character` 上增加资源点反向绑定字段
3. 重写资源点懒结算流程，接入“跨天结算 + 当日刷新 + 过期清理”
4. 实现玩家单点绑定与跨点自动迁移
5. 实现无主点占领与有主点宗门级对抗
6. 实现多个进攻宗门分开记进度的逻辑
7. 实现 `输运` 产出加成
8. 调整宗门 / 资源点 UI 展示
9. 补测试与验收

## 16. 验收标准

一期验收至少应满足：

1. 玩家可创建、加入、退出宗门
2. 宗门职位能按贡献自动生成标签
3. 正道 / 魔道玩家只能加入对应阵营宗门
4. 中立玩家可加入任意宗门
5. 资源点按自然日随机刷新，并在存在 `3` 天后枯竭消失
6. 不额外设计全服资源点活跃上限，依然能自然稳定循环
7. 玩家同一时间只能绑定 `1` 个资源点岗位，切换时会自动从旧点移除
8. 无主点需要 `2` 次占领，有主点需要 `4` 次占领
9. 多个进攻方按宗门分别计算进度，不能合并为一个总攻方
10. 已占有资源点的争夺会触发宗门级对抗判定，而不是纯比较谁高一点
11. 战力相近时不会出现“只差一点就固定必赢”的情况
12. `输运` 不参与攻防，只按每名成员 `+10%` 提高该点产出，且单点不设上限
13. 灵矿 / 魂脉 / 灵泉能按类型发放不同奖励
14. 灵泉奖励按“固定修为 + 当前阶段上限 5%”发放，并受剩余可得修为约束
15. 悬赏讨伐可同时发放器魂与灵石

## 17. 后续预留

本规格明确为后续扩展保留空间，但一期不提前实现：

- 宗门审批
- 宗门权限
- 宗门仓库
- 宗门建筑
- 宗门科技
- 灵石商店
- 宗门拍卖 / 交易
- 资源点特殊地形
- 宗门联盟 / 联合进攻
- 更复杂的宗门战赛季系统

一句话：

> **先把资源点做成“动态刷新、有寿命、有攻守、有后勤、有多人宗门对抗感”的轻量玩法，再考虑更重的宗门战争外壳。**
