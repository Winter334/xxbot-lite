# 01-角色与基础成长规格

> 状态：Draft v1
> 作用：定义角色成长框架、境界结构、基础三维、本命法宝与强化规则。
> 关联文档：
> - `design/discord-xianbot-v1-dev-guide.md`
> - `design/specs/01a-境界基础属性表.md`
>
> 本文档定义“规则层”。具体境界数值表单独放在 `01a`，避免本文件变成数值垃圾山。

---

## 1. 文档范围

本规格只处理以下内容：

1. 角色成长主轴（境界 / 小境界）
2. 角色基础三维（杀伐 / 护体 / 身法）
3. 本命法宝的一期形态
4. 强化规则与限制
5. 与基础成长强相关的核心字段

本规格**不**处理以下内容：

- 挂机修炼结算公式
- 气机恢复公式
- 通天塔强度与首通奖励
- 论道榜挑战规则
- 战斗伤害精确公式
- 播报模板

这些将在后续规格中单独定义。

---

## 2. 设计原则

### 2.1 总原则

- 境界是主线成长轴，数值上必须占主导。
- 本命法宝提供长期养成感，但不能压过境界。
- 命格是小幅偏移，不是决定胜负的唯一因素。
- 角色基础三维必须全部是“长期稳定有效”的属性，不能出现某一主属性在后期明显失效的情况。

### 2.2 战力占比目标

一期目标占比：

- **境界基础属性：约 80%**
- **本命法宝强化属性：约 15%**
- **命格 / 称号 / 其他杂项：约 5%**

该占比是后续数值反推的重要约束。若后续具体公式与此目标明显偏离，应以此目标为准进行修正。

## 3. 境界结构

### 3.1 大境界列表

一期开放以下 9 个大境界：

1. 炼气
2. 筑基
3. 结丹
4. 元婴
5. 化神
6. 炼虚
7. 合体
8. 大乘
9. 渡劫

### 3.2 小境界结构

每个大境界固定分为 4 个小境界：

1. 前期
2. 中期
3. 后期
4. 圆满

### 3.3 显示规则

默认显示格式建议为：

- `炼气·前期`
- `元婴·圆满`

允许在表现层降级为：

- `炼气前期`
- `元婴圆满`

**注意：** 表现层可变，逻辑层不可变。内部逻辑必须始终使用结构化字段，而不是解析显示文本。

### 3.4 内部结构化表示

建议使用以下字段：

```text
realm_key       = lianqi | zhuji | jiedan | yuanying | huashen | lianxu | heti | dacheng | dujie
realm_index     = 1..9
stage_key       = early | mid | late | perfect
stage_index     = 1..4
global_stage_index = (realm_index - 1) * 4 + stage_index   // 1..36
```

建议逻辑判断优先基于：

- `realm_index`
- `stage_index`
- `global_stage_index`

而不是直接比较中文字符串。

## 4. 角色基础三维

### 4.1 三维定义

一期只保留 3 个核心主属性：

- **杀伐**：稳定输出主属性
- **护体**：稳定生存主属性
- **身法**：高波动主属性，影响先手、暴击、闪避、暴击伤害

### 4.2 基础三维来源

角色总三维由以下部分相加得到：

```text
total_atk = base_atk(realm, stage) + artifact_atk_bonus + fate_atk_bonus + other_atk_bonus
total_def = base_def(realm, stage) + artifact_def_bonus + fate_def_bonus + other_def_bonus
total_agi = base_agi(realm, stage) + artifact_agi_bonus + fate_agi_bonus + other_agi_bonus
```

其中：

- `base_*` 来自 `01a-境界基础属性表.md`
- `artifact_*_bonus` 来自本命法宝强化累计值
- `fate_*_bonus` 来自命格（若命格为对应属性或间接加成）
- `other_*_bonus` 为后续系统预留，一期可为 0

**建议：**
- `base_*` 不需要落库存储，可在运行时按境界查表得到。
- `total_*` 也不建议落库，优先运行时计算，避免冗余状态失真。

### 4.3 身法的职责定义

身法在一期中承担以下职责：

1. 决定先手倾向
2. 影响暴击率（有上限）
3. 影响闪避率（有上限）
4. 影响暴击伤害（无硬上限）

### 4.4 身法相关硬性边界

一期拍板如下：

- **暴击率上限：60%**
- **闪避率上限：30%**
- **暴击伤害：无硬上限**

设计原因：

- 暴击率与闪避率有上限，防止身法变成无脑堆叠最优属性。
- 暴击伤害无硬上限，确保身法在后期继续成长，不会因为前两项接近上限而沦为废属性。
- 身法玩家天然具备更高波动性：上限更高、下限更低。

### 4.5 约束说明

由于身法同时影响多个战斗维度，后续战斗规格在设计公式时必须遵守：

- 身法相关系数应当保守
- 不得让同等数值的身法期望收益明显碾压杀伐或护体
- 一期内不得再给身法附加额外功能（例如连击、反击、额外回合、命中修正等）

## 6. 本命法宝系统（一期）

### 6.1 系统定位

一期每个角色只有 **1 件本命法宝**。

该法宝是角色的永久成长载体，用来承接：

- 强化成长
- 后续词条 / 洗炼扩展
- 面板展示与归属感

### 6.2 一期明确规则

- **不掉落其他法宝**
- **不做法宝品质**
- **不做法宝背包**
- **不做法宝替换**
- **不做法宝吞噬 / 狗粮系统**
- 全程只培养这一件法宝

### 6.3 名字规则

法宝名字采用：

> **预设随机 + 允许一次改名**

具体规则：

1. 玩家首次创建角色时，系统从预设法宝名池中随机分配一个名称
2. 玩家在后续任意合适时机可进行 **1 次免费改名**
3. 改名机会用掉后，一期内不可再次改名

### 6.4 改名限制（轻量）

一期仅保留最低限度限制：

- 长度限制建议：2 ~ 12 个字符
- 禁止换行
- 禁止 `@everyone` / `@here`
- 不做复杂敏感词系统

说明：

- 怪名称不是一期主要问题，不必为此引入复杂命名审核系统。
- 若后续公共频道污染明显，再额外补规则。

## 7. 法宝强化系统（一期）

### 7.1 强化资源

强化唯一资源固定为：

> **器魂**

一期不引入其他强化材料分支。

### 7.2 强化上限

强化等级上限与大境界绑定：

> **每提升 1 个大境界，强化上限 +10**

即：

- 炼气：最高 `+10`
- 筑基：最高 `+20`
- 结丹：最高 `+30`
- 元婴：最高 `+40`
- 化神：最高 `+50`
- 炼虚：最高 `+60`
- 合体：最高 `+70`
- 大乘：最高 `+80`
- 渡劫：最高 `+90`

说明：

- 强化上限只看大境界，不区分前中后圆满
- 此设计用于保证“境界压着法宝走”

### 7.3 强化成功率分段表

一期采用固定分段表：

- `+0 ~ +9`：100%
- `+10 ~ +19`：80%
- `+20 ~ +29`：60%
- `+30 ~ +39`：50%
- `+40+`：25%

说明：

- 不使用复杂公式
- 不做额外强化保护符
- 不做保底、幸运值、失败累计增益

### 7.4 强化失败规则

强化失败时：

- 只损失本次消耗的器魂
- **不掉级**
- **不碎法宝**
- **不锁强化**
- **不产生负面状态**

### 7.5 强化成功后的成长方式

强化成功时：

1. 先得到一笔 `reinforce_growth_total`（总成长值）
2. 再把这笔总成长值随机分配给：
   - 杀伐加成
   - 护体加成
   - 身法加成

### 7.6 总成长值的来源

一期暂不单独制作“强化成长值独立表”，而采用以下原则：

> **强化总成长值应根据境界基础属性表反推，确保法宝总贡献长期接近约 15% 的目标。**

也就是说：

- 先确定 `01a` 的境界基础三维表
- 再根据目标占比推导各强化区间的大致成长值
- 不允许强化成长与境界基础属性脱节

### 7.7 随机分配规则（一期建议）

一期建议使用最简单的实现：

- 把 `reinforce_growth_total` 视为若干离散点数
- 逐点均匀随机分配到 `atk / def / agi`
- 三项都允许本次获得 0 点

说明：

- 该方案实现简单
- 能自然制造玩家法宝分化
- 不需要额外词条或复杂分配策略

后续若觉得波动过大，可在 v2 中改为“至少分配到 2 项”的保守规则，但一期先不加。

## 8. 战力显示值

### 8.1 是否显示

一期 **显示总战力值**。

理由：

- 玩家容易感知成长
- 适合晒面板
- 挑战论道榜时具备参考意义

### 8.2 设计原则

战力值是：

- 展示性综合指标
- 可用于排序参考
- 但**不是战斗胜负的唯一依据**

尤其由于身法流派存在更强波动性：

- 高战力并不保证必胜
- 低战力也可能因身法波动实现逆转

### 8.3 公式归属

战力显示公式将在后续文档中定义，但必须满足：

- 输入只来自总三维与基础成长系统
- 不得绕开基础三维额外创造一套平行属性体系

## 10. 非目标 / 禁止实现

在本规格下，以下内容一律视为非目标：

- 多装备位
- 法宝掉落池
- 法宝品质系统
- 法宝背包
- 法宝吞噬 / 狗粮
- 复杂命名审核
- 暴击词条 / 闪避词条 / 洗炼系统
- 技能系统
- 元素克制
- 职业路线
- 强化保护符 / 幸运值 / 强化保底
- 基于未来扩展而提前做的通用强化框架

一句话：

> **一期只做“境界成长 + 一件法宝 + 简单强化 + 三维分化”，不要额外长系统。**
