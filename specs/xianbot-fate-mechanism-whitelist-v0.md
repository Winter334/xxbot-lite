# 修仙 BOT 命格系统新增机制白名单 v0

## 一、目的

本白名单用于约束命格系统的扩展方向，避免命格设计脱离现有项目实现能力，或为了少数命格效果引入过重、过散的底层机制。

目标是：
- 允许少量新增通用机制，让命格真正有“命”的味道
- 不让命格系统膨胀成复杂的技能、状态或天赋树系统
- 让新增机制能够被多个命格复用，而不是只服务单个效果

---

## 二、总体原则

1. **命格不是纯战斗被动，也不是单一资源词条**  
   命格应体现角色在战斗、成长、得失、波动中的整体命运倾向。

2. **优先新增通用钩子，不新增一次性特例逻辑**  
   若某个效果只能服务 1 个命格，则原则上不为其单独扩底层。

3. **命格可做双刃剑效果**  
   命格允许同时包含正面与负面效果，以体现“命数有偏、福祸相依”的设计方向。

4. **命格应优先复用现有系统**  
   新增机制应建立在已有战斗结算、资源结算、面板展示能力上，而非新造大系统。

---

## 三、允许新增的机制白名单

## 1. 统一资源结算命格修正层

### 说明
在已有的资源结算节点前后，增加一层统一的命格修正逻辑，用于对资源结果进行倍率、保底、波动等修正。

### 适用范围
- 挂机 / 闭关修为结算
- 爬塔修为结算
- 爬塔器魂结算
- 游历修为结算
- 游历器魂结算
- 后续悬赏器魂结算
- 后续劫掠器魂结算

### 支持的命格方向
- 修为收益偏高 / 偏低
- 器魂收益偏高 / 偏低
- 收益稳定型
- 收益波动型
- 双刃剑型（如修为更高，但器魂更低）

### 复杂度
**低**

### 推荐优先级
**高**

---

## 2. 统一战斗触发点

### 说明
为命格开放少量通用战斗触发点，用于挂接简单条件型效果。触发点数量应控制在少数核心阶段，不做多层复杂状态机。

### 建议触发点
- `battle_start`：战斗开始时
- `before_attack`：出手前
- `after_hit`：命中 / 受击后
- `low_hp`：低血量状态下
- `battle_end`：战斗结束时

### 支持的命格方向
- 开局强势型
- 低血爆发型
- 稳定防守型
- 条件增伤 / 减伤型
- 极端命格 / 双刃剑命格

### 复杂度
**中低**

### 推荐优先级
**高**

---

## 3. 波动性与保底修正

### 说明
允许命格修改收益或战斗结果的波动区间、最低保底、极端结果倾向，以体现“命好 / 命硬 / 命薄 / 福祸相依”的体验差异。

### 支持方向
- 收益更稳定
- 收益更容易偏高
- 收益更看脸
- 最低收益保底
- 战斗发挥更稳定
- 极端高收益 / 极端低收益更易出现

### 适用范围
- 修为结算
- 器魂结算
- 战斗单次伤害表现
- 战斗结果波动

### 复杂度
**中**

### 推荐优先级
**中高**

---

## 4. 双刃剑命格模板

### 说明
允许单个命格同时带有一个正面效果和一个负面效果，不额外新增特殊底层机制，仅通过组合通用 effect 实现。

### 支持方向
- 高收益 / 高风险
- 高伤害 / 高承伤
- 高器魂 / 低修为
- 稳成长 / 弱战斗
- 强战斗 / 弱资源

### 复杂度
**低**

### 推荐优先级
**高**

---

## 四、暂不建议新增的机制

以下机制当前版本原则上不纳入命格系统首发实现范围：

### 1. 多层状态叠加机制
例如：叠层、充能、层数转换、按层爆发。

**原因：** 状态管理复杂，容易将命格演化为技能系统。

### 2. 主动释放型命格技能
例如：点击施放命格、消耗能量发动效果。

**原因：** 会明显改变交互模型，偏离“命格”而转向“主动技能”。

### 3. 命格专属资源 / 能量条
例如：命格值、天命槽、宿命值。

**原因：** 额外增加系统复杂度和 UI 成本。

### 4. 多命格槽位
例如：主命格 + 副命格 + 命星组合。

**原因：** 当前已确定玩家仅持有 1 个命格，首发版本不应扩成组合系统。

### 5. 单个命格专属的特例底层
例如：为了某一条命格单独新增只会被它自己使用的判断链。

**原因：** 维护成本高，后续扩展会失控。

### 6. 直接影响气运获取
例如：闭关气运加成、悬赏气运加成、改命返还气运。

**原因：** 气运当前核心职责是“逆天改命货币”，不宜让命格系统反向自我喂养。

---

## 五、首发命格建议依赖的机制组合

为保证“风味足够”且“实现不失控”，首发命格建议优先从以下组合中产生：

### 组合 A：资源结算修正 + 双刃剑模板
适合：
- 福缘深厚
- 贪狼入命
- 厚积薄发
- 祸福相依

### 组合 B：静态属性修正 + 战斗触发点
适合：
- 七杀临身
- 无垢道体
- 洞虚灵目
- 狂骨

### 组合 C：波动修正 + 保底模板
适合：
- 命途多舛
- 福薄如纸
- 天命所归

## 六、命格设计方向与强度基线（当前共识）

### 1. 命格设计主次
首发命格的设计方向，当前按以下主次分配：

#### 主力方向
- **成长倾向型**
- **收益结构型**
- **波动命数型**

#### 点缀方向
- **属性倾向型**
- **战斗风格型**

设计意图：
- 命格首先决定角色“怎么成长、怎么得、怎么亏、命顺不顺”
- 其次才决定其“打架时是什么风格”
- 避免命格系统整体滑向“战斗天赋树 / 神通被动”

---

### 2. 命格效果原则

1. **尽量不做一次性效果**  
   不优先采用“首次出手生效”“首次受击生效”“一场战斗仅触发一次”这类命格效果。

2. **尽量不做临时效果**  
   不优先采用“前 2 回合生效”“持续 3 回合”“临时 Buff / Debuff”这类命格效果。

3. **优先做常驻型、结构型、倾向型效果**  
   命格应长期影响角色的成长方式、资源结构、战斗气质与得失倾向，而不是在某个瞬间短暂亮一下。

4. **命格效果必须写成明确规则**  
   后续设计命格时，应尽量明确写出：
   - 作用对象
   - 触发时机
   - 具体倍率 / 结构变化
   - 实现位置

禁止使用“更稳”“更顺”“少有失手”“更容易偏高”这类不可直接落地的抽象表述，除非进一步拆解成具体规则。

---

### 3. 命格强度原则

命格必须是强效果，不能只是边角料级别的小词条。

设计要求：
- 不做 **3% / 5%** 这种存在感过低的加成
- 命格应当明显改变角色的成长路线、资源结构、战斗底色或风险偏好
- 玩家获取命格后，应能明显感知并描述其风格，例如：
  - 我这命格偏修为
  - 我这命格偏器魂
  - 我这命格偏高风险高收益
  - 我这命格特别稳

---

### 4. 命格强度基线

#### 4.1 成长型命格
建议常用强度档位：
- **+15%**
- **+20%**
- **+25%**

适用方向：
- 修为获取加成
- 器魂获取加成
- 某类通用成长结果加成

#### 4.2 属性型命格
建议常用强度档位：
- **单属性型：+12%**
- **双属性型：+10% / +10%**
- **三属性型：+8% / +8% / +8%**

适用方向：
- 杀伐 / 护体 / 身法
- 以及其组合型命格

#### 4.3 双刃剑型命格
建议常用结构：
- **+25% / -15%**（成长 / 资源类）
- **+15% / -10%**（属性倾向类）
- **+20% / 承伤 +10%**（战斗风格类）

设计要求：
- 正面收益必须足够大，值得玩家记住
- 负面代价必须真实存在，形成“命偏了”的感觉

#### 4.4 波动型命格
建议常用改动幅度：
- **波动区间变化 50%**
- 或使用明确规则，例如：
  - 最低收益保底提高
  - 高收益结果权重显著提高
  - 收益上限提高、下限降低

设计要求：
- 不允许只写“更稳 / 更赌”
- 必须拆解为可直接实现的概率 / 区间 / 保底结构变化

---

### 5. 首发命格池配比建议
当前建议命格池总量为 **34 条**，按以下配比分布：

- **成长型：8**
- **属性型：8**
- **双刃剑型：8**
- **波动命数型：6**
- **战斗风格型：4**

设计意图：
- 主力是成长 / 属性 / 双刃剑，保证首发命格池有足够存在感
- 波动命数型负责提供“命味”
- 战斗风格型少量点缀，不让命格整体滑向纯战斗系统

---

## 七、当前结论

命格系统首发版本允许在现有项目基础上新增少量通用机制，重点是：

1. **统一资源结算命格修正层**
2. **统一战斗触发点**
3. **波动性与保底修正**
4. **双刃剑命格模板**

并在此基础上进一步遵守以下设计共识：

5. **命格以成长倾向、收益结构、波动命数为主，以属性倾向与战斗风格为辅**
6. **命格尽量不做一次性、临时性效果**
7. **命格必须是强效果，不做 3% / 5% 级别的小词条**
8. **命格效果必须能写成明确规则，不允许停留在抽象描述层**

这样可以保证：
- 命格不只是旧系统换皮
- 命格有足够风味与区分度
- 实现范围可控，不会演化成另一套大型系统
- 后续继续扩具体命格时不会再次跑偏
